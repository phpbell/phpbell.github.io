Odeio CSS

Quando comecei a mexer com HTML por volta do ano 2004 o CSS já era algo complexo, acessível apenas a uma elite de programadores.

As incompatibilidades entre os diferentes browsers tornavam necessária a utilização de programas como o TopStyle que mostravam quais browsers suportavam cada um dos seletores.

O fato é que com o tempo o CSS tornou-se cada vez mais complexo e eu acabei odiando essa linguagem cada vez mais.

Atualmente, com a onda do design responsivo devem existir pelo menos uma meia dúzia de formas diferentes para se construir o layout básico de um site, todas com nomes bonitinhos: tableless, flexbox, grid, bootstrap grid, 960 grid system, holy grail layout, etc.

Odeio profundamente tudo isso, sinto como se a internet tivesse involuindo ao adicionar suporte a resoluções de tela tão baixas como a dos smartphones.

CSS-in-JS

Outra onda que surgiu mais recentemente foi a do CSS-in-JS (JSX). O causador dessa onda foi provávelmente o React do Facebook.

O React nada mais é do que uma gambiarra criada pelo Facebook para tentar aproximar seus aplicativos mobile do código nativo dos smartphones.

Como efeito colateral disso um bando de developers começaram a usar essa gambiarra para fazer sites e PWAs. A web nunca esteve tão poluída e pesada por conta do CSS-in-JS quanto está agora.

Não sei de onde veio a idéia de que colocar o CSS dentro do JS ou até virtualizar o DOM melhoraria a performance dos sites, só sei que browsers mais antigos como o Firefox estão tendo dificuldade de otimizar o carregamento de todas essas novas gambiarras absurdas que surgem a cada dia.
